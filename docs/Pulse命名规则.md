# Pulse 命名规则（M0 基线）

> 本文件是 Pulse 重构期间的统一命名基线。后续代码、文档、仓库命名均以此为准。

## 1) 统一命名

| 维度 | 约定值 | 说明 |
|---|---|---|
| 品牌名 | `Pulse` | 对外展示名称（README、文档、演示） |
| Python 包名 | `pulse` | 代码命名空间（`src/pulse/`） |
| CLI 命令 | `pulse` | 统一入口（如 `pulse start`） |
| 建议仓库名 | `pulse-agent` | 推荐用于 GitHub 仓库，降低重名冲突 |
| Docker/服务名 | `pulse` | compose service、容器标签保持一致 |

## 2) 禁用旧命名

- 新增代码和文档中，不再使用 `OfferPilot`、`Jarvis` 作为主名称。
- 历史调研资料可保留旧名称，不强制回写（例如 `docs/V2架构设计.md`）。

## 3) 文件与目录命名约定

- 新代码目录统一使用小写：`pulse`、`core`、`modules`、`tools`。
- 文档文件优先使用 `Pulse` 前缀：如 `Pulse架构方案.md`、`Pulse实施计划.md`。
- 对外命令示例统一写法：`pulse start`、`pulse doctor`（后续若新增命令同规则）。

## 4) 执行优先级

1. 先统一代码命名空间（包名/CLI/import）。
2. 再统一仓库名与发布名。
3. 最后处理历史目录名（本地文件夹名可延后，不阻塞开发）。

## 5) 变更控制

- 命名规则若需修改，必须先更新本文件，再执行批量重命名。
- 重命名提交建议独立为单独 commit，避免与功能改动混杂。
