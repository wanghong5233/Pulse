# 工程宪法

## 0. 第一性原理(顶层总则)

每写一行代码 / 测试 / 注释前,问自己**两个问题**:

1. 这个问题的**本质**是什么?
2. 最**简单正确**的解法是什么?

不因"别人这么写""大模型这么建议""先跑起来再说""看起来更专业"绕过这两步。

---

## 1. 代码宪法

1. **暴露优于兜底**
   异常该处理(网络抖动 / 用户输入)就处理,该暴露(配置缺失 / 代码 bug)就 fail-fast;**不** 用 `except Exception`、`a or b or c or default`、`return None` 伪造正常结果掩盖问题。

2. **源头修,禁止下游堆补丁**
   脏数据在**入口洗一次**,不在多处反复 normalize;上游形态不稳定就修上游或收敛到单一 adapter;`# HACK` / `# workaround` 必须有 issue 追踪根因。

3. **抽象服从需求,不服从想象**
   只为**当前需求**设计;不为"将来可能用到"留参数 / 配置项 / 泛型 / 单实现抽象类。YAGNI 优先于 SOLID。

4. **命名与结构优于注释**
   如果代码需要注释解释"在做什么",先改**代码**;注释只写"为什么"(见 §3)。

**反模式索引**(出现任一即停):防御式逃避、补丁式兼容、过度设计、魔法数字、硬编码配置、日志打完整密钥、吞异常无日志。

---

## 2. 测试宪法

1. **测行为,不测实现**
   断言"输入 → 输出 / 副作用 / 可观察状态",**不** 断言调用次数 / 私有方法 / 内部字段 / 刚写的常量(同源复读)。测试改量 > 代码改量时,测的是实现,**重写**。

2. **用真实 fixture,不脑补形态**
   payload 从真实 trace / provider 响应切片存成 fixture,**不** 手搓 dict(多 / 少字段、类型不符、带异常码点等)。

3. **投入产出比:契约 > 集成 > 单元 > e2e**
   首选**集成 / 合同测试**(一条真实调用链),其次纯函数单元测试;**不** 为覆盖率补边角、**不** 测第三方库的已知行为。

4. **真实 trace / 真机路径是最后一道闸**
   全绿 ≠ 功能对;合入前对真实用户/真实环境路径至少验收一次。

5. **一批改动一次验证**
   禁止"改一行跑一遍";那是焦虑的心理按摩,不是严谨。

**反模式索引**:同源复读、形态造假、过度 mock、测第三方、per-field unit、绿色但无声、表演式重跑。

---

## 3. 注释宪法

1. **只写代码无法表达的信息**
   反直觉的 why、不变式、陷阱、历史教训、**外部稳定**契约出处(标准、权威规范)。

2. **不搬规范**
   权威文档**长文**不进代码;一个文件在 **module 级 docstring 一处** 给出出处即够。

3. **短而可维护**
   长 docstring 挤占注意力;签名与结构已说清的,不重复。注释与实现冲突时,以**实现**为准,删过时注释。

**反模式索引**:复述实现、长文规范搬运、礼貌式空泛 docstring、用注释写未来路线图、在注释里做“被谁引用”的倒排索引、在代码里引会漂移的**章节号**当真相来源。

---

## 4. 系统形态 / 工程形态

> 与 §1 的分工:§1 管**单段代码**对不对;§4 管**整条系统**是否可配、可测、可观测、可关、可交付。

1. **配置有单一真源** — 环境、开关、特性经统一入口进入系统;**不** 在业务层散落读环境或魔法常量。

2. **依赖显式、边界清晰** — 可替换;领域逻辑**不**依赖隐式全局状态或不可测的单例(除非框架硬性约束,且范围极小)。

3. **可观测成体系** — 关键路径、异常路径、对外边界有**一致**的观测方式;秘钥与敏感信息**不**以明文进入日志/遥测的默认通道。

4. **生命周期可管理** — 并发与长驻/调度工作有**可持有、可取消、可等到结束**的句柄;退出时按序释放资源(连接、子任务、句柄、锁)。

5. **合入 = 可维护** — 不夹带为调试而写的死代码、大块注释、无跟踪的 `TEMP` 开关(有则必须能指向 issue 与移除条件)。

**反模式索引**:业务里散读环境、print 当日志、无协调的 fire-and-forget 后台、关机不等人、全量秘钥/PII 进默认日志、领域代码绑死全局可变单例、调试用临时代码永久合入
